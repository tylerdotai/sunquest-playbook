## ⚡ What Is This?

A **machine-verified, fully-mapped playbook** for SunQuest RPG — an image-map browser RPG with no visible stats, no client-side state, and a purely server-stateful game engine.

This repo contains:
- The **complete world graph** — every URL, link, and route in the game
- **Combat outcome model** — P(win) per enemy type, derived from statistical sampling
- **Item/location encyclopedia** — what every place does, what you'll find, and what to do next
- **Winning walkthrough** — step-by-step route to complete the game
- **God-mode agent code** — the automation scripts used to generate this playbook

## 🎯 Combat System

**SunQuest is a stateless server-stateful game.** There is no visible health bar, gold counter, or stat display in the UI. The server tracks your state between pages — the browser only renders the current HTML scene.

### Combat Outcome Model

| Enemy Type | Location | P(Win) | Notes |
|---|---|---|---|
| Skeleton | 714-2 (floor 1) | ~50% | Early dungeon; deaths common |
| Dungeon enemies | skull-dungeon-2 | TBD | Deeper floor |
| Crypt enemies | /the-crypt/ | TBD | Unknown difficulty |
| World enemies | Various | TBD | Unsurveyed |

*Combat outcomes are sampled empirically. See [combat-model.md](combat/combat-model.md) for full data.*

### How Combat Works

1. Navigate to a combat page
2. Click the fight link (JS `document.getElementById('link-N').click()`)
3. Server resolves outcome server-side
4. You land on either a **victory page** (loot/gold) or **death page** (game over)
5. From death: return to last safe location and retry
6. No health bar to track — HP is an internal counter that resets on death

### Death Strategy

When you die, you restart at the game's defined respawn point. Key strategy: **always know your nearest safe haven before engaging combat.**

## 🏆 Winning the Game

**Goal:** Navigate the full world, complete all major quest lines, and reach the final ending.

### Optimal Path Strategy

```
1. START → Fast Travel Map (/starting-location/s-map/)

3. GATHER → Find Potion (/findpotion/) — get item
```

## 🔧 God-Mode Agent

The Hermes Agent automation scripts used to build this playbook are in [`agent/`](agent/).

### Architecture

- **Browser automation** — cua-driver for macOS GUI control
- **Storage inspector** — localStorage/sessionStorage/cookie audit (confirmed: all empty — purely server-side state)
- **URL crawler** — breadth-first graph traversal of all game pages
- **Combat sampler** — runs same combat link N times, logs outcome distribution
- **Outcome model** — P(win) per enemy type from statistical reversal

### Key Scripts

| Script | Purpose |
|---|---|
| `crawl.js` | BFS URL crawler — maps every link in the game |
| `combat_sampler.js` | Run same combat link N times, log win/loss URL |
| `storage_audit.js` | Snapshot localStorage/sessionStorage/cookies before/after actions |
| `link_mapper.js` | Dump all links from current page as JSON |

### Storage Audit Results

```
localStorage:  0 keys on every page
sessionStorage: 0 keys on every page
cookies: 0 keys on every page
IndexedDB: empty

→ SunQuest is 100% server-stateful. No client-side state exists.
→ The browser is a stateless HTML renderer.
→ Combat can only be modeled statistically.
```
